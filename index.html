SIGGRAPH Asia 2026 Workshop

Overview

The second Volumetric Video Challenge focuses on two practical problems for immersive media systems: efficient compression and sparse-view reconstruction.

Compression Track

Compress dynamic 4DGS-style reconstructions while preserving rendering quality under an average budget of 200 KB/frame.

Sparse-View Track

Reconstruct dynamic volumetric humans from 6 calibrated training views. This year all tracks reconstruct full RGB images with background.

Evaluation is performed on full RGB images, including foreground and background regions.

Winning teams' technical reports will be included in the SIGGRAPH Asia workshop proceedings.

Challenge Tracks

The challenge features two tracks focusing on compression and sparse-view reconstruction for volumetric videos.

Compression Track

Compress dynamic 4DGS-style models under an average 200 KB/frame budget while maintaining high-quality full-image rendering.

  • Input: calibrated multi-view RGB videos
  • Baseline: baseline method will be provided
  • Output: rendered held-out test views, model, scripts, report

Sparse-View Track

Reconstruct dynamic volumetric humans from 6 calibrated training views and render held-out test views.

  • Input: 6-view calibrated RGB videos
  • Output: rendered held-out test views and report

Dataset

The VVC26 dataset contains calibrated dynamic multi-view RGB videos. All methods reconstruct and render complete RGB scenes with background.

Compression Track

Training RGB videos, train/test camera parameters, time offsets, and baseline assets will be released for each sequence.

Sparse-View Track

Only 6 calibrated training views are released for reconstruction. Held-out test-view camera parameters are provided for rendering.

Compression Track Structure

sequence/
├── images/              # training-view RGB frames
│   ├── 00/000000.jpg
│   └── ...
├── train_intri.yml      # training camera intrinsics
├── train_extri.yml      # training camera extrinsics
├── test_intri.yml       # held-out test camera intrinsics
├── test_extri.yml       # held-out test camera extrinsics
├── time_offsets.json    # camera/frame synchronization
└── baseline/            # baseline assets, if provided

Sparse-View Track Structure

sequence/
├── images/              # 6 training-view RGB frames
│   ├── 00/000000.jpg
│   └── ...
├── train_intri.yml      # 6-view training intrinsics
├── train_extri.yml      # 6-view training extrinsics
├── test_intri.yml       # held-out test camera intrinsics
├── test_extri.yml       # held-out test camera extrinsics
└── time_offsets.json    # camera/frame synchronization

Evaluation

Metrics: PSNR, SSIM, and LPIPS on official held-out test views.

Region: Full RGB image evaluation, including foreground and background regions.

Compression Track

Rank = [rank(PSNR) + rank(SSIM) + rank(LPIPS)] / 3

A submission is valid only if its content-dependent representation is smaller than an average 200 KB/frame. Invalid submissions are excluded from the ranking.

Size counts all content-dependent files required for rendering. Shared content-agnostic code or pretrained backbones may be excluded after organizer review.

Sparse-View Track

Rank = [rank(PSNR) + rank(SSIM) + rank(LPIPS)] / 3

Entries are ranked by full-image reconstruction quality under the 6-view input setting. The test views are held out and evaluated by the organizers.

Detailed Requirements

Compression Track

Provided Data

  • Training-view RGB videos or frames.
  • Training and held-out test camera parameters.
  • Time offsets or synchronization metadata when needed.
  • Baseline assets and rendering script, if released.

Requirements

  • Final representation must be below an average 200 KB/frame.
  • Render full RGB images for all official test views.
  • Submit model artifacts, rendering scripts, environment file, rendered results, and a draft technical report.
  • Winning teams submit final ACM-formatted technical reports for inclusion in the SIGGRAPH Asia workshop proceedings.

Sparse-View Track

Provided Data

  • 6 calibrated training-view RGB videos or frames.
  • Training camera intrinsics and extrinsics for the 6 views.
  • Held-out test camera intrinsics and extrinsics for rendering.
  • Time offsets or synchronization metadata when needed.

Requirements

  • Use only the provided sparse training views for reconstruction.
  • Render full RGB images for all official test views.
  • Submit rendered results and a draft technical report.
  • Winning teams submit final ACM-formatted technical reports for inclusion in the SIGGRAPH Asia workshop proceedings.
